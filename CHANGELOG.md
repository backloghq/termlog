## [Unreleased]

### Added
- Initial repo scaffold (package.json, tsconfig, eslint, vitest, src/index.ts).
- **Posting list codec (`src/codec.ts`)** — `encodeVByte`/`decodeVByte` (7-bit continuation, handles >32-bit values via `Math.floor` division and multiplication to avoid signed-integer overflow), `encodePostings`/`decodePostings` (delta-encoded doc IDs + VByte tf; format: `vbyte(count) | [vbyte(delta) vbyte(tf)]*`), `postingIterator` (lazy decode for query path). 19 tests covering round-trip identity, delta math, edge cases (empty, single entry, large IDs up to 2^35, tf=0/65535), partial iterator advance, and size sanity.
- **Term dictionary v0.1 (`src/term-dict.ts`)** — `TermDict` class with sorted array of `(term, postingsOffset, postingsLength, df)` entries; `add`, `lookup` (binary search, O(log n)), `serialize`/`deserialize` (compact binary: uint32 count + per-entry uint16 termLen + UTF-8 term bytes + three uint32 fields), `fromMap` (sorts unsorted map for segment writer). 20 tests: empty dict, single/many entries, first/middle/last binary search, 1000-entry dict, Unicode terms (café, 東京), 100-entry round-trip, `fromMap` sort correctness.
- **StorageBackend abstraction (`src/storage.ts`)** — `StorageBackend` interface (`readBlob`, `writeBlob`, `listBlobs`, `deleteBlob`, optional `isLocalFs`); `FsBackend` implementation with atomic writes (`.tmp` + rename), prefix-filtered `listBlobs` (root-flat, no recursive walk), idempotent `deleteBlob`, recursive directory creation. 12 tests: round-trip, atomic write (no stale `.tmp`), overwrite, missing-file errors, prefix filtering, empty-dir and no-match list, delete idempotency, nested dirs, interface compat (FsBackend + mock S3-shape backend).
- **Lint script extended to cover `tests/`** — `eslint src/ tests/`.
- **Segment writer + reader (`src/segment.ts`, `src/crc32.ts`)** — `SegmentWriter` accumulates `addPosting`/`setDocLength` calls in any order, sorts by term and doc ID, then `flush(id, backend)` writes a single binary `.seg` file atomically (`.seg.tmp` → rename via backend). `SegmentReader.open` verifies CRC32 for all three regions (postings, sidecar, dict), throws `SegmentCorruptionError` naming the corrupted region on mismatch, and exposes `lookupTerm`, `postings` (lazy iterator), `decodePostings` (full materialization), `docLen`, `terms` (sorted generator). File layout: `[postings region][doc-length sidecar][term dictionary][52-byte footer with magic, version, offsets, CRC32s]`. Pure-JS CRC32 in `src/crc32.ts` (IEEE 802.3 polynomial, table-driven). 14 tests: round-trip (basic corpus + 100-term), doc lengths, termCount/docCount, df correctness, posting iterator vs full decode, missing term iterator, sorted terms, out-of-order addPosting, CRC corruption detection for postings/footer/dict regions.
- **SegmentManager + manifest (`src/manager.ts`)** — `SegmentManager` class with write buffer, auto-flush at configurable threshold, atomic manifest (`manifest.tmp` → rename), and reader snapshot isolation. `SegmentManager.open` reads an existing manifest and reopens all referenced `SegmentReader`s. `add(docId, terms[])` buffers docs; `flush()` writes a new segment via `SegmentWriter`, increments the monotonic `generation` counter, and replaces the reader snapshot immutably so callers holding a prior snapshot are unaffected. `segments()` returns the current immutable snapshot; `commitGeneration()` returns the monotonic counter. Manifest format: `{ version, generation, segments[], tokenizer, totalDocs, totalLen }`. 16 tests: fresh-index open, empty-buffer flush no-op, generation monotonicity, segments-per-flush growth, term correctness in flushed segments, bufferedCount reset, auto-flush at threshold, reader snapshot isolation, manifest persistence across reopen (generation, segment count, totalDocs, totalLen, postings), manifest.tmp cleanup, many-docs-per-segment packing.
- **Compaction — LSM one-tier merge (`SegmentManager.compact()`)** — K-way merge of all snapshot segments by (term, docId); doc IDs re-numbered to a dense range; doc lengths carried forward. Writes merged segment atomically, then atomic manifest swap removing old segment IDs and adding the merged segment (segments flushed during compaction are preserved). Old segment files deleted post-commit. `mergeThreshold` option (default 8) triggers auto-compact after each auto-flush. 14 tests: no-op on 0/1 segments, N-to-1 merge, all terms present, tf values preserved, doc lengths preserved, generation increment, 8-segment query-equivalence vs single-segment baseline, manifest.tmp cleanup, old-file deletion, reopen after compact, reopen after compact+flush, reader snapshot isolation pre/post compact, auto-trigger at mergeThreshold.
- **Query iterators + boolean ops (`src/query.ts`)** — `SegmentPostingIter` wraps a single segment's lazy posting iterator with `advance()` and `seek(docId)` (linear forward scan; skip-list is v0.2). `MultiSegmentIter` k-way merges per-segment iters for one term in docId order, summing tf across segments sharing a docId. `andQuery` (zigzag merge) yields only docIds present in all term iterators; `orQuery` (k-way union) yields every docId in any term iterator with per-term tfs collected. Both return `Generator<QueryPosting>` where `QueryPosting = { docId, tfs: Map<term, tf> }`. 27 tests: SegmentPostingIter (iterate, empty, seek-exact, seek-between, seek-past-end, advance-on-exhausted); MultiSegmentIter (single seg, two-seg merge, missing term, seek, no-segs); andQuery (two-term intersection, empty intersection, single term, multi-segment, three terms, no-terms, no-segs, sorted order); orQuery (union, disjoint sets, multi-segment, single term, missing term, no-terms, no-segs, sorted order).
- **BM25 scoring layer (`src/scoring.ts`)** — `bm25Score(tf, dl, df, N, k1, b, avgdl)` pure function mirrors agentdb's formula exactly (`idf = log((N-df+0.5)/(df+0.5)+1)`, `norm = k1*(1-b+b*(dl/avgdl))`, `score = idf*tf*(k1+1)/(tf+norm)`); `avgdl=0` falls back to 1. `BM25Ranker` wraps `orQuery` and emits `{docId, score}` sorted score-desc, tie-broken by string-lexicographic docId asc (matches agentdb's string-ID sort so "11"<"2"). 29 tests across four suites: pure-function properties (tf=0, very-common term, avgdl=0 fallback, manual formula, tf/df/dl monotonicity); ranker edge cases (empty terms, N=0, missing term, sorted order, lex tie-breaking, limit); 10-query parity suite against agentdb's TextIndex.searchScored on a shared 50-doc corpus at `k1=1.2 b=0.75`, `k1=0 b=0.75`, `k1=1.2 b=0`, and `k1=1.2 b=1` — all scores match to 10 decimal places.
- **Crash recovery (`src/manager.ts`, `tests/crash-recovery.test.ts`)** — `SegmentManager.open` now handles all 5 failure modes: (1) orphan `*.seg.tmp` deleted on open; (2) unreferenced `*.seg` files (segment written, manifest not updated) deleted on open; (3) stale `manifest.tmp` from interrupted rename deleted before reading `manifest.json`; (4) segment CRC corruption propagated as `SegmentCorruptionError`; (5) manifest JSON parse failure throws `ManifestCorruptionError`. New `ManifestCorruptionError` class exported from `src/manager.ts`. 8 tests: orphan .seg.tmp cleanup, orphan .seg cleanup, manifest.tmp + manifest.json coexistence, manifest.tmp with no manifest.json (fresh-index path), SegmentCorruptionError propagation, ManifestCorruptionError on bad JSON and truncated JSON, round-trip reopen consistency.
- **Concurrency tests (`tests/concurrency.test.ts`)** — verifies reader-snapshot isolation semantics under interleaved async flush and compact. 10 tests: pre-flush snapshot unchanged after flush; multiple flush snapshots each see their own generation; Promise.all racing read + flush; pre-compact snapshot queries correctly after compact deletes old `.seg` files (data is eagerly loaded into Buffer on `SegmentReader.open`); Promise.all racing read + compact; segments added during compact are preserved in post-compact view; manifest consistency on reopen after flush+compact cycles; many-flush reopen sees all committed docs; racing adds then flush lands in manifest. **Bug found and fixed**: concurrent `writeBlob("same-path")` calls shared the same `<path>.tmp` temp file, causing ENOENT on rename when one call's temp was overwritten by another. Fixed in `FsBackend.writeBlob` by using a nanosecond-precision nonce: `<path>.<hrtime>.tmp`.
- **Stress test (`tests/stress.test.ts`)** — gated behind `STRESS=1`; defaults to 10k docs in normal CI. 7 assertions: every flush segment ≤ `flushThreshold` docs; post-compact segment count = 1; merged segment `docCount` = N; vocab term df > 0 for all terms; specific doc IDs retrievable; BM25 OR query p95 ≤ 50ms (500ms under STRESS=1); heap growth ≤ 100MB (800MB under STRESS=1). Smoke-tested at 50k docs: 61ms indexing, compacts to 1 segment.
- **Benchmark suite (`tests/bench.test.ts`)** — gated behind `BENCH=1`; skips gracefully otherwise. Builds 10K and 100K indexes, measures: indexing throughput (docs/sec), query latency p50/p95/p99 for 1-/2-/5-term OR BM25 queries, compaction cost (wall-clock + bytes before/after), reopen latency. Prints a formatted console table. 8 relative-invariant assertions (no hardware-dependent absolutes): 100K throughput ≥ 10K×0.5; latency p99 ≤ p50×10 per term-count; 5-term ≥ 1-term cost; compaction reduces segment count; merged docCount correct; reopen < indexing time. On AMD Ryzen 7 9700X: 10K→430K docs/sec, 100K→1.1M docs/sec; 1-term BM25 p99 57ms, 5-term p99 246ms (100K corpus, 100 pre-compact segments); compact 260ms, reopen 8ms.
